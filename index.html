Tipos de pseudo-linguagens

  • de Projeto de Algoritmos - divisão de um problema em subproblemas
    • formato livre
  • De implementação/executável
    • linguagens como VisualAlg, utilizada para aprendizado, mas com sintaxe rígida, podendo ser executada na implementação de problemas simples

Pseudocódigos de projeto são utilizados durante toda a carreira como programador;

Pseudocódigos executáveis são utilizados somente nos primeiros meses de aprendizado.

Any problem in computer science can be solved with another layer of indirection. But that usually will create another problem.

  • David Wheeler, criador do conceito de subrotinas (subprogramas)

Subprogramas: procedimentos e funções

procedimento le_vetor(vetor, tamanho_vetor)
inicio

fimprocedimento
procedimento le_vetor(var v: vetor de inteiro, var n: inteiro)
var
  i : inteiro;
inicio
  leia (n)
  para i de 1 ate n faca
     leia (v[i])
  fimpara
fimprocedimento

Mesma interface, outra implementação...

procedimento le_vetor(var v: vetor de inteiro, var n: inteiro)
var
  x : inteiro
inicio
  n <- 0
  leia (x)
  enquanto x <> 0 faça
     n <- n + 1
     v[n] <- x
     leia (x)
  fimenquanto
fimprocedimento
procedimento imprime_vetor(vetor, tamanho_vetor)
inicio
  //imprima cada elemento do vetor, das posições de 1 a n, separados por espaço
  //imprima nova linha, ao final
fimprocedimento
procedimento imprime_vetor(var v: vetor de inteiro, var n: inteiro)
var
  i : inteiro
inicio
  para i de 1 ate n faca
     escreva (v[i], "")
  fimpara
  escreval ("")
fimprocedimento

Conclusões

  • Pseudo-linguagens facilitam o aprendizado e a comunicação dos programadores
  • Use a técnica de refinamentos sucessivos top-down no projeto de algoritmos
  • Funções e procedimentos separam "o quê fazer?" do "como fazer?"
    • além de permitir reuso de código
    • e o gerenciamento da complexidade
